# 기능 명세서 — institution-ui

**문서 버전:** 1.0
**작성일:** 2026-04-15

## 1. 개요

본 문서는 institution-ui의 화면 단위 기능·요청/응답 계약·상태 전이·에러 처리·키보드 동작·와이어프레임을 정의한다.

## 2. 액터

| 액터 | 설명 |
|------|------|
| Anonymous | 미인증 사용자. 로그인/회원가입만 접근 |
| USER | 기관·유형 조회만 가능 |
| ADMIN | 전체 CRUD 및 관리 |

## 3. 인증 기능

### 3.1 로그인

**경로:** `/login`
**접근:** Public

**와이어프레임:**
```
+--------------------------------------+
|                                      |
|         [로고]                       |
|      연구기관 관리                   |
|                                      |
|   Username  [_________________]      |
|   Password  [_________________]      |
|                                      |
|            [  로그인  ]              |
|                                      |
|   계정이 없으신가요? 회원가입       |
+--------------------------------------+
```

**요청:**
```http
POST /api/auth/login
Content-Type: application/json

{ "username": "admin", "password": "admin1234" }
```

**응답 (성공, 200):**
```json
{
  "status": 200,
  "message": "로그인 성공",
  "data": { "username": "admin", "role": "ADMIN" }
}
```

**UI 동작:**
- 성공: 인증 상태 저장 후 `/institutions`로 이동
- 401 (INVALID_CREDENTIALS): "자격 증명이 올바르지 않습니다" 알림 표시, 폼 유지
- 400 (VALIDATION_ERROR): 필드별 인라인 메시지
- 500: "서버 오류가 발생했습니다" 알림

**키보드:**
- Enter: 폼 제출
- Tab: 필드 간 이동

### 3.2 회원가입

**경로:** `/signup`
**접근:** Public

**와이어프레임:**
```
+--------------------------------------+
|   [로고]  회원가입                   |
|                                      |
|   Username  [_________________]      |
|             4-50자                   |
|   Password  [_________________]      |
|             최소 8자                 |
|   Role      ( ) USER  ( ) ADMIN      |
|                                      |
|            [ 가입하기 ]              |
|                                      |
|   이미 계정이 있으신가요? 로그인    |
+--------------------------------------+
```

**요청:**
```http
POST /api/auth/signup
Content-Type: application/json

{ "username": "user01", "password": "user1234", "role": "USER" }
```

**응답 (성공, 201):**
```json
{ "status": 201, "message": "가입 완료", "data": { "id": 5, "username": "user01", "role": "USER" } }
```

**UI 동작:**
- 성공: `/login`으로 이동 + "가입이 완료되었습니다. 로그인해 주세요." 알림
- 409 (DUPLICATE_USERNAME): username 필드에 "이미 사용 중인 사용자명입니다"
- 400: 필드별 인라인 메시지

### 3.3 로그아웃

**트리거:** 네비게이션 바 "로그아웃" 버튼

**요청:**
```http
POST /api/auth/logout
X-XSRF-TOKEN: <token>
```

**UI 동작:**
- 성공: 클라이언트 인증 상태 초기화 → `/login` 이동

### 3.4 라우트 보호

**동작 순서:**
1. 보호 페이지 진입 시 클라이언트가 `GET /api/auth/status` 호출
2. 200 + `{ authenticated: true, username, role }` → 렌더링
3. 401 → `/login`으로 리다이렉트
4. ADMIN 전용 페이지에서 role !== ADMIN → `/institutions` 리다이렉트 + "접근 권한이 없습니다" 알림

## 4. 네비게이션 바

**표시 조건:** 인증된 페이지 (로그인/회원가입 제외)

**와이어프레임:**
```
+------------------------------------------------------------------+
| [로고] 연구기관 관리   기관 목록  기관 유형       admin [ADMIN] 로그아웃 |
+------------------------------------------------------------------+
```

**요소:**
- 좌측: 로고 + "연구기관 관리" (클릭 시 `/institutions`)
- 중앙: "기관 목록", "기관 유형" 링크 (현재 경로 하이라이트)
- 우측: 사용자명, role 뱃지 (USER=회색, ADMIN=파란색), 로그아웃 버튼

## 5. 기관 관리 기능

### 5.1 기관 목록

**경로:** `/institutions`

**와이어프레임:**
```
+------------------------------------------------------------------+
|  기관 목록                                        [+ 기관 등록] |
+------------------------------------------------------------------+
| 기관명          | 코드       | 유형     | 대표자   | 설립일     |

| 한국과학기술원 | KAIST     | 대학    | 홍길동  | 1971-02-16 |
| ETRI            | ETRI      | 정출연  | 김철수  | 1976-12-30 |
| ...                                                              |
+------------------------------------------------------------------+
```

**요청:** `GET /api/institutions`

**응답 (200):**
```json
{
  "status": 200,
  "message": "조회 성공",

        "code": "UNIVERSITY",
        "name": "대학",
        "description": "4년제 대학·대학원 포함",
        "sortOrder": 1,
        "active": true,
        "createdAt": "2026-04-01T09:00:00",
        "updatedAt": "2026-04-01T09:00:00"
      },
      "address": "대전광역시 유성구 ...",
      "phone": "042-350-2114",
      "email": "info@kaist.ac.kr",
```

**UI 동작:**
- 행 클릭: `/institutions/{id}`로 이동
- 빈 배열: "등록된 기관이 없습니다" 안내
- ADMIN: 우측 상단 "+ 기관 등록" 버튼 표시
- USER: 등록 버튼 비표시

### 5.2 기관 상세

**경로:** `/institutions/[id]`

**와이어프레임:**
```
+------------------------------------------------------------------+
| ← 목록으로                                    [수정] [삭제]   |
|                                                                  |
|  한국과학기술원 (KAIST)                                        |
|                                                                  |
|  유형        대학                                               |
|  주소        대전광역시 유성구 ...                              |
|  전화        042-350-2114                                       |
|  이메일      info@kaist.ac.kr                                   |
|  홈페이지    https://kaist.ac.kr                                |
|  대표자      홍길동                                             |
|  설립일      1971-02-16                                         |
|                                                                  |
|  등록일      2026-04-15 10:30                                   |
|  수정일      2026-04-15 10:30                                   |
+------------------------------------------------------------------+
```

**요청:** `GET /api/institutions/{id}`

**UI 동작:**
- 404: "기관을 찾을 수 없습니다" + 목록으로 돌아가기 버튼
- ADMIN: 우측 상단 "수정", "삭제" 버튼
- USER: 액션 버튼 비표시

### 5.3 기관 등록

**경로:** `/institutions/new`
**접근:** ADMIN

**와이어프레임:**
```
+------------------------------------------------------------------+
|  기관 등록                                                       |
|                                                                  |
|  기관명 *     [__________________]                              |
|  코드 *       [__________________]  (A-Z, 0-9, _, -)            |
|  유형 *       [ 대학           ▼ ]                              |
|  주소 *       [__________________]                              |
|  전화         [__________________]                              |
|  이메일       [__________________]                              |
|  홈페이지     [__________________]                              |
|  대표자       [__________________]                              |
|  설립일       [ YYYY-MM-DD      ]                               |
|                                                                  |
|                              [ 취소 ]  [ 등록 ]                  |
+------------------------------------------------------------------+
```

**요청:**
```http
POST /api/institutions
Content-Type: application/json
X-XSRF-TOKEN: <token>

{
  "name": "한국과학기술원",
  "code": "KAIST",
  "institutionTypeId": 1,
  "address": "대전광역시 유성구 ...",
  "phone": "042-350-2114",
  "email": "info@kaist.ac.kr",
  "website": "https://kaist.ac.kr",
  "representative": "홍길동",
  "establishedDate": "1971-02-16"
}
```

**UI 동작:**
- 성공 (201): 상세 페이지로 이동 + "등록되었습니다" 알림
- 409 (DUPLICATE_INSTITUTION_CODE): code 필드에 "이미 사용 중인 기관 코드입니다"
- 400 (INSTITUTION_TYPE_INACTIVE): 유형 필드에 "비활성화된 기관 유형은 사용할 수 없습니다"
- 400 (VALIDATION_ERROR): 필드별 인라인 메시지
- 취소 버튼: `/institutions`로 이동

**폼 검증 (클라이언트):**
| 필드 | 검증 |
|------|------|
| 기관명 | 필수, 최대 200자 |
| 코드 | 필수, 최대 50자, `^[A-Z0-9_-]+$` |
| 유형 | 필수 선택 |
| 주소 | 필수, 최대 500자 |
| 전화 | 선택, 최대 30자 |
| 이메일 | 선택, 이메일 형식 |
| 홈페이지 | 선택, URL 형식 |
| 대표자 | 선택, 최대 100자 |
| 설립일 | 선택, YYYY-MM-DD |

### 5.4 기관 수정

**경로:** `/institutions/[id]/edit`
**접근:** ADMIN

**UI 동작:**
- 진입 시 `GET /api/institutions/{id}`로 기존 값 로드 후 폼 채움
- 제출: `PUT /api/institutions/{id}` (전체 필드 교체)
- 성공: 상세로 이동 + "수정되었습니다"
- 409/400: 등록과 동일 처리

### 5.5 기관 삭제 (확인 모달)

**트리거:** 상세 페이지 "삭제" 버튼 클릭

**와이어프레임:**
```
  +----------------------------------------+
  |  기관 삭제 확인                    [X] |
  +----------------------------------------+
  |  삭제를 진행하려면 아래 값을 정확히    |
  |  입력해 주세요.                        |
  |                                        |
  |  기관 코드  [__________________]       |
  |  기관명     [__________________]       |
  |                                        |
  |  삭제된 기관은 복구할 수 없습니다.     |
  |                                        |
  |              [ 취소 ]  [ 삭제 ]        |
  +----------------------------------------+
```

**요청:**
```http
DELETE /api/institutions/{id}
Content-Type: application/json
X-XSRF-TOKEN: <token>

{ "code": "KAIST", "name": "한국과학기술원" }
```

**UI 동작:**
- 성공 (200): 모달 닫기 → `/institutions` 이동 + "삭제되었습니다"
- 409 (DELETE_MISMATCH): 모달 내 "삭제 확인 정보가 일치하지 않습니다"
- ESC: 모달 닫기
- 배경 클릭: 모달 닫기

**경로:** `/institution-types`

**와이어프레임:**
```
+------------------------------------------------------------------+
|  기관 유형                                     [+ 유형 등록]    |
```

**접근:** ADMIN


**UI 동작:**
- 성공: 상세로 이동 + 알림

**경로:** `/institution-types/[id]/edit`
**접근:** ADMIN

**트리거:** 상세 페이지 "비활성화" 버튼

**모달:**
```
  +----------------------------------------+
  |  유형 비활성화 확인                [X] |
  +----------------------------------------+
  |  "대학" 유형을 비활성화합니다.         |
  |                                        |
  |  비활성화 후:                          |
  |  - 신규 기관 등록 시 선택 불가         |
```

## 7. 알림 & 에러 표시 규칙

| 상황 | 표시 방식 |
|------|-----------|
| 필드 검증 실패 | 필드 하단 빨간 텍스트 |
| API 400 (필드별) | 해당 필드 하단 매핑 |
| API 401 | 즉시 `/login` 리다이렉트 |
| API 403 | 페이지 상단 빨간 배너 + 리다이렉트 |
| API 404 | 페이지 전체 빈 상태 컴포넌트 |
| API 409 | 상황별 필드 또는 모달 에러 |
| API 500 | 페이지 상단 빨간 배너 (자동 소멸 없음) |
| 성공 | 페이지 상단 초록 배너, 3초 자동 소멸 |

## 10. CSRF 처리

모든 상태 변경 요청(POST/PUT/DELETE)에 `X-XSRF-TOKEN` 헤더를 자동 주입한다.

**동작 순서:**
1. 서버가 응답 시 `XSRF-TOKEN` 쿠키 설정 (HttpOnly=false)
2. `lib/csrf.ts`가 `document.cookie`에서 값 파싱
3. `lib/api.ts` fetch 래퍼가 요청 헤더에 `X-XSRF-TOKEN: <token>` 포함
4. 예외 경로: `POST /api/auth/signup`, `POST /api/auth/login`

**검증:**
- 토큰 누락 시 403 ACCESS_DENIED
- `credentials: "include"` 필수 (쿠키 전송)
